# 🌀 Фрактальный Навигатор (Fractal Navigator Bot)

Telegram-бот для процедурной генерации, автоматической рассылки и интерактивного исследования математических фракталов на комплексной плоскости. Проект разработан с использованием гибридного вычислительного ядра **PyTorch (CUDA)** / **NumPy (CPU)**.

В основе концепции лежит отказ от предобученных генеративных моделей в пользу чистого математического рендеринга. Каждое изображение формируется в реальном времени путем итерационного решения динамических систем, уравнения которых конструируются процедурным генератором.

---

## 🛠 Ключевые архитектурные и технические решения

### 1. Двухкомпонентный вычислительный движок (Hybrid Engine)
* **PyTorch CUDA**: При наличии совместимого графического процессора NVIDIA вычисления сетки выполняются параллельно на тензорных ядрах. Для исключения дедлоков при инициализации выполняется прогрев CUDA-контекста на главном потоке.
* **NumPy CPU**: При отсутствии графического ускорителя вычисления автоматически переносятся на центральный процессор с использованием векторизованных матричных операций NumPy.
* **Предотвращение переполнений**: Для предотвращения генерации `NaN` и `Infinity` при вычислении сверхстепенных или экспоненциальных функций применяется жесткое ограничение вещественной и мнимой частей аргументов в диапазоне `[-15.0, 15.0]`, а также регуляризационная добавка $\epsilon = 10^{-20}$ в операциях деления и логарифмирования.

### 2. Процедурный компилятор формул (AST -> RPN)
* Бот генерирует случайные абстрактные синтаксические деревья (AST), включающие тригонометрические функции, сигмоиды, экспоненты, инверсию, сопряжение и комплексные константы.
* Дерево транслируется в стек **обратной польской записи (RPN)** для выполнения на стековой виртуальной машине.
* Реализован собственный синтаксический анализатор на базе алгоритма **сортировочной станции Дийкстры (Shunting-yard)**, который преобразует введенные пользователем инфиксные формулы (например, `(Z^2) + C`) в валидный RPN-стек с поддержкой унарных операций.

### 3. Интеллектуальное позиционирование и эстетический фильтр
* **Поиск границ (Boundary Search v2.0)**: Для нахождения наиболее детализированных областей на каждом шаге зума вычисляется градиент яркости. К карте итераций применяется двумерный быстрый фильтр скользящего среднего (`fast_uniform_filter` размером 15x15) для построения фазовой маски границ фрактала. Точка следующего приближения выбирается случайным образом среди кандидатов с наивысшим градиентно-фазовым индексом (Aesthetic Attraction Index).
* **Адаптивный тональный маппинг**: Спектр итераций сжимается с помощью комбинации динамической гамма-коррекции (вычисляемой на основе медианы яркости изображения) и логарифмического преобразования, что исключает появление «выжженных» или слишком темных областей.
* **Многофакторная валидация качества (Aesthetic Filter)**:
  1. *Площадь тела фрактала*: Проверяется соотношение заполненных и пустых пикселей (допустимый диапазон: от 1.5% до 65%).
  2. *Контрастность*: Стандартное отклонение яркости пикселей вне тела фрактала должно быть не менее `0.12`.
  3. *Фильтр шума (Корреляция)*: Вычисляется коэффициент корреляции Пирсона между соседними пикселями по горизонтали и вертикали. Кадры с выраженным высокочастотным шумом (корреляция $< 0.45$) отбраковываются.
  4. *Фильтр шума (Total Variation)*: Оценивается средняя полная вариация яркости. Превышение порога `0.60` сигнализирует о хаотичной зернистости изображения.
  5. *Детектор монотонных линий*: Оценивается сонаправленность градиентов $dX$ и $dY$ после размытия. Если корреляция градиентов превышает `0.70` (что характерно для простых параллельных полос), кадр бракуется.

### 4. Многопоточность и управление ресурсами
* **Управление GIL**: При генерации сеток с большим числом итераций вычислительный цикл каждые 10 шагов приостанавливает поток на `0.005` секунды (`time.sleep`), добровольно уступая GIL сетевым потокам Telegram-пуллинга, что предотвращает зависание интерфейса бота во время тяжелого рендеринга.
* **Контроль утечек памяти**: После экспорта графики в буферы `BytesIO` (JPEG с SSAA-сглаживанием и оригинальный PNG) тяжелые объекты принудительно удаляются из памяти, вызывается сборщик мусора `gc.collect()` и очищается кэш видеопамяти `torch.cuda.empty_cache()`.
* **Потокобезопасность**: Доступ к дисковым операциям логирования, рендеринга и работы с файлами настроек разграничен блокировками `threading.Lock`.

### 5. Отказоустойчивость и планировщик
* **SQLite База данных**: Используется класс-менеджер `StatsManager` со встроенными блокировками записи для безопасной работы из параллельных потоков.
* **Защита от дубликатов при рассылке**: Таблица `broadcast_deliveries` использует составной первичный ключ `(epoch, chat_id)`. Это гарантирует, что пользователь не получит один и тот же авто-фрактал дважды в один и тот же интервал времени, даже если планировщик будет перезапущен.
* **Каскадный Fallback при авто-рассылке**: Если за 15 попыток процедурный генератор не смог построить эстетически валидный фрактал, планировщик снижает разрешение рендеринга и пробует снова. Если и это не удается, активируется аварийный режим со стабильной красивой координатой глубокого зума множества Мандельброта. Если отправка невозможна, пользователи получают уведомление о временных технических работах.
* **Наблюдатель активности (Heartbeat)**: Бот содержит фоновый поток, регулярно отправляющий пинг-запросы на внешний хостинг, поддерживающий актуальный статус доступности в профиле бота (`Online` / `Offline`).

## 📂 Структура проекта

```text
├── bot.py             # Главный файл запуска бота, обработчики Telegram-команд
├── config.py          # Инициализация переменных окружения, логгера, сетевого слоя и локализации
├── database.py        # Управление БД SQLite, подписчиками, настройками и лимитами пользователей
├── math_engine.py     # Парсер инфиксных формул, компилятор AST в RPN и вычислительные ядра
├── renderer.py        # Генерация сеток, адаптивный маппинг, фильтры эстетики и экспорт
├── scheduler.py       # Автоматическая рассылка (каждые 2 часа) и каскадные режимы генерации
├── subscribers.txt    # Список ID активных подписчиков на рассылку
├── user_settings.json # Сохраненные предпочтения пользователей (язык, масштаб)
└── bot_stats.db       # База данных статистики и доставок рассылки
```

## 🚀 Быстрый запуск

### 1. Установка базовых зависимостей
```bash
git clone https://github.com/MHP-Fan/fractal-navigator-bot.git
cd fractal-navigator-bot
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Установка PyTorch (с поддержкой CUDA)
Для корректной работы аппаратного ускорения на видеокартах NVIDIA установите соответствующую сборку PyTorch. Пример команды для CUDA 12.1:
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

### 3. Настройка окружения
Создайте файл `.env` в корневой директории:
```env
BOT_TOKEN=1234567890:ABCdefGhIJKlmNoPQRsTUVwxyZ
ADMIN_ID=987654321
```

### 4. Запуск
```bash
python bot.py
```
