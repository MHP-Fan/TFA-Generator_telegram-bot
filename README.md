### 1. Рекомендуемая структура файлов

Создайте в корневой папке вашего проекта следующие файлы:

```text
fractal-abyss-bot/
│
├── bot.py                  # Основной локальный код бота (ваш service.py)
├── requirements.txt        # Список зависимостей для установки
├── .gitignore              # Список файлов, которые нельзя загружать в GitHub
├── README.md               # Главное описание проекта для вашей страницы на GitHub
│
└── cloud/                  # Папка со скриптами для облачного наблюдателя
    ├── flask_app.py        # Flask-сервер для PythonAnywhere
    └── check_status.py     # Скрипт проверки статуса для крона
```

### 3. Файл `requirements.txt`
Этот файл позволит любому человеку (или вам при переносе на другой ПК) установить все зависимости одной командой: `pip install -r requirements.txt`.

```text
pyTelegramBotAPI>=4.12.0
numpy>=1.24.0
matplotlib>=3.7.0
requests>=2.31.0
pillow>=9.5.0
# Примечание: PyTorch (torch) обычно устанавливается вручную под конкретную версию CUDA, 
# поэтому его лучше не вносить в общий requirements, а описать установку в README.
```

### 4. Оформление страницы проекта: `README.md`
Это «лицо» вашего проекта на GitHub. Ниже представлен готовый, структурированный шаблон на русском языке. Он написан в меру техническим, понятным языком без лишней рекламы, подчеркивая архитектуру вашего кода.

Создайте файл `README.md` и скопируйте туда этот разметку Markdown:

```markdown
# 🌀 Хаб Фрактальной Бездны (Fractal Abyss Bot)

Telegram-бот для процедурной генерации математических фракталов на комплексной плоскости с использованием аппаратного ускорения **NVIDIA CUDA (PyTorch)**. 

Проект разработан как альтернатива современному генеративному ИИ — здесь нет нейросетей («нейрослопа»), картинок из интернета или готовых шаблонов. Каждый пиксель — это результат сотен численных итераций по формулам, сгенерированным случайным образом на лету.

---

## 🚀 Архитектурные особенности

* **Процедурная грамматика (AST -> RPN)**: Бот генерирует случайные деревья абстрактного синтаксиса (AST), содержащие тригонометрию, сигмоиды, экспоненты и комплексные числа, преобразует их в обратную польскую нотацию (RPN) и рендерит на GPU.
* **Двойной вычислительный движок**: При наличии видеокарты вычисления идут параллельно на тензорных ядрах через **PyTorch CUDA**. Если GPU недоступен, вычисления плавно переносятся на CPU через оптимизированные матрицы **NumPy**.
* **Адаптивный тональный маппинг**: Спектральные значения итераций логарифмируются и корректируются динамической гаммой, чтобы избежать «выжженных» пикселей и выдать глубокую контрастную картинку.
* **Фильтр эстетического качества**: Алгоритм анализирует сложность и распределение градиентов полученной матрицы, автоматически отбраковывая скучные однотонные заливки или серый шум.
* **Умный трекер прогресса**: Пользователь видит живой статус генерации и шагов зума прямо в сообщении Telegram без перегрузки лимитов API (Rate Limits).
* **«Кнопка мертвеца» (Dead Man's Switch)**: Система облачного мониторинга на базе связки Flask (на бесплатном *PythonAnywhere*) и *cron-job.org*. Если на домашнем ПК выключится свет или пропадет интернет, бот автоматически переключит свой статус в Telegram-профиле на `🔴 Вне сети`.

---

## 🛠️ Системные требования

## 📦 Быстрый старт (Локально)

### 1. Клонирование репозитория и установка зависимостей
```bash
git clone https://github.com/yourusername/fractal-abyss-bot.git
cd fractal-abyss-bot
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для Linux:
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Установка PyTorch с поддержкой CUDA
Для аппаратного ускорения генерации на GPU Nvidia перейдите на [официальный сайт PyTorch](https://pytorch.org/get-started/locally/) и выполните команду установки под вашу версию CUDA. Пример для CUDA 12.1 (Windows):
```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

### 3. Настройка токена
Вы можете задать токен бота через переменные окружения:
* **Windows**: `set BOT_TOKEN=ваш_токен_от_botfather`
* **Linux**: `export BOT_TOKEN="ваш_токен_от_botfather"`

Или временно вставить его напрямую в переменную `BOT_TOKEN` внутри файла `bot.py` (не забудьте убрать его перед коммитом в Git!).

### 4. Запуск бота
```bash
python bot.py
```

## ☁️ Развертывание системы статуса «В сети / Офлайн»

Инструкции по настройке независимого внешнего наблюдателя, защищающего статус вашего бота от внезапного выключения света или сбоя провайдера:

1. Загрузите файлы из папки `/cloud` на бесплатный хостинг **PythonAnywhere**.
2. В файле `flask_app.py` укажите ваш токен бота.
3. Перейдите на **cron-job.org**, зарегистрируйте бесплатный аккаунт и настройте опрос вашего эндпоинта `https://ваш_логин.pythonanywhere.com/check` каждые 2–3 минуты.
4. В локальном файле `bot.py` на вашем ПК убедитесь, что функция `heartbeat_loop` отправляет запросы на ваш адрес `https://ваш_логин.pythonanywhere.com/ping`.

### Что сделать перед загрузкой на GitHub?
1. Переименуйте ваш текущий файл бота в `bot.py` (или укажите его имя в README.md).
2. Вынесите токен бота из кода в переменную окружения или убедитесь, что перед загрузкой в репозиторий в поле `BOT_TOKEN` стоит пустая заглушка `"ВАШ_ТОКЕН_СЮДА"`. Помните, что GitHub сканирует коммиты на наличие токенов и мгновенно блокирует утекшие ключи в целях безопасности!
